Overview

RIFT is a personal home server project aimed at centralizing storage, services, and experimentation within one modular platform. It replaces scattered local drives and third-party tools with a controlled environment that can grow over time.

The system is treated as infrastructure rather than a single-purpose build. Core priorities include reliability, scalability, and ease of maintenance so that new services can be added without disrupting existing workloads.

Throughout the project the focus stays on practical systems engineering lessons: storage planning, service isolation, controlled network access, and long-term maintainability.

Problem / Goals / Requirements

Problem

  • Data and media were scattered across multiple devices and drives
  • Third-party services limited control over availability and configuration
  • No centralized platform existed for experimenting with self-hosted tools

Goals

  • Centralize storage and core services inside a single system
  • Provide reliable media access and organized file management
  • Create a flexible platform for future services and automation work

Requirements

  • Modular design that supports future expansion
  • Redundant storage to protect critical data
  • Simple maintenance and update process

System / Architecture

RIFT combines storage, compute, and network management inside a modular enclosure. Storage pools are segmented to isolate media, personal files, and project datasets so that each can be scaled or backed up independently.

Services are deployed through containers, allowing individual workloads to be updated, replaced, or paused without impacting the remainder of the system. This isolation keeps experimentation from jeopardizing core functions.

Networking, access controls, and monitoring scripts are configured with a focus on clarity rather than complexity to ensure the platform remains maintainable over the long term.
