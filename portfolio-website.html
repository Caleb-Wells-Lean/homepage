Overview

This project involved the design and development of a personal engineering portfolio website intended to serve as a central reference for projects, experience, and technical growth. The site was built to present work clearly and consistently while remaining easy to update over time.

Rather than relying on static resumes or slide decks, the website was structured to support living project documentation, allowing detailed case studies, visuals, and supporting artifacts to be accessed directly. Emphasis was placed on clarity, maintainability, and a professional presentation suitable for both technical and non-technical audiences.

Development was completed in short, focused work sessions over time, allowing design decisions to mature between iterations. Throughout the project, priority was given to clean structure, readable content, and ease of updating information as new projects are added.

Problem / Goals / Requirements

Problem

  • Static resumes and decks do not capture evolving technical projects
  • Updating project information required reworking entire documents
  • Existing formats lacked structured space for detailed documentation

Goals

  • Create a repeatable case-study format for all projects
  • Provide clear access to deliverables and visuals
  • Keep navigation simple and intuitive

Requirements

  • Static implementation using HTML, CSS, and minimal JavaScript
  • Mobile-friendly and readable across devices
  • Easy to update without complex tooling

System / Architecture

The website was built as a modular collection of reusable sections, allowing content to be added or updated without redesigning the overall layout. Core pages share a consistent structure to ensure uniform presentation across all projects.

Project content is organized using a standardized case-study format, separating overview, execution, deliverables, and outcomes. This mirrors common engineering documentation and improves readability for technical reviewers.

The site relies on a lightweight, text-first approach to maintain fast load times and long-term maintainability. Interactive elements are kept minimal and only appear where they improve clarity or navigation.
